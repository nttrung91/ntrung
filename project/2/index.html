t/n

Portfolio

t/n is a portfolio site built to reflect on my current design and development experience.


Design

The goal of this project focuses on delivering a fun and engaging experience when viewing the site. The first step is to determine the core interaction between objects of the site. How do components move and behave? What interaction is engaging and unique? In my opinion, defining the interaction model allows me to visualize how the site behaves at the end.

After defining the interaction model, I begin working on the layout’s architecture and feature. Which content goes where? What happen when this component is hovered?

When the layout’s architecture is finished, I begin designing the look and feel of the layout.


Development

The goal of the development process emphasizes on code’s maintainability and productivity.

Focusing on code’s maintainability means I use a set architecture, mixture of OOCSS, SMACSS, and Atomic CSS frameworks to structure out my code. By using this architecture, it allows me to quickly make changes and create components on the page. Additionally, I also maintain a live Styleguide, built using Hologram, to act as a reference for all the elements on the site.

By focusing on productivity, I always want to find tools to speed up the development process. I use Grunt build tool to help automate tasks and Sublime Text’s Plugins (e.g. Emmet) to help write code more quickly
